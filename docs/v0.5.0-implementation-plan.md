# v0.5.0 implementation plan

## Outcome

v0.5.0 adds two optional, additive contract blocks: `architecture` for
structured repository design guidance and `agents` for consistent operating
constraints. Both flow through validation, normalization, analysis, and all
five generated adapters without changing the output of existing v0.4.0
contracts.

The package remains pre-1.0, the contract remains `version: 1`, and every new
surface follows ADR-0009's additive compatibility policy.

## Accepted scope

- ADR-0032: optional `architecture.boundaries`, `architecture.invariants`, and
  `architecture.key_decisions`.
- ADR-0033: optional `agents.disallowed_actions`,
  `agents.approval_required_for`, and `agents.context_files`.
- Path-form and duplicate validation for decision and context files.
- `agent-ready analyze` existence and Markdown-file checks for both path lists.
- Shared rendering into AGENTS.md, CLAUDE.md, Cursor, Copilot, and Gemini
  outputs with Markdown-safe text.
- A v2 adapter compatibility corpus containing unchanged v0.4 cases and new
  v0.5 cases.

## Explicit non-goals

- Runtime enforcement of agent guidance.
- Import-graph or architecture-boundary enforcement; that remains v0.7 work.
- `allowed_tools`, `default_instructions`, `quality_gates`, or `handoff`
  contract blocks.
- Environment variables, working directories, network access, LLM calls, or
  telemetry.
- Contract `version: 2`.

## Implementation sequence

### Increment 1 — `0.5.0-alpha.1`: architecture domain

1. Add the optional `architecture` JSON Schema definitions with all size and
   shape limits from ADR-0032.
2. Extend raw and normalized contract types.
3. Normalize list order and decision paths deterministically.
4. Add semantic validation for path safety and duplicate decision files.
5. Render the Architecture section through the shared adapter renderer.
6. Add schema, normalization, semantic, escaping, adapter, and byte-identity
   regression tests.

### Increment 2 — `0.5.0-alpha.2`: agent guidance domain

1. Add the optional `agents` JSON Schema definitions from ADR-0033.
2. Extend raw and normalized contract types.
3. Normalize context paths while preserving declared order.
4. Add semantic validation for path safety and duplicates.
5. Render Agent Constraints through the shared adapter renderer.
6. Add schema, normalization, semantic, escaping, adapter, and byte-identity
   regression tests.

### Increment 3 — `0.5.0-beta.1`: analysis and compatibility

1. Generalize the existing instruction-source analysis path so decision and
   context files use the same bounded filesystem boundary.
2. Add stable diagnostics for missing or invalid decision/context files and
   document them in the diagnostic reference and `explain` registry.
3. Create `compatibility/adapter-output/v2`, retaining all v1 cases and adding
   complete architecture-only, agents-only, and combined cases.
4. Update the complete example contract and all five golden adapter fixtures.
5. Update the contract reference, CLI reference, adoption guide, API stability
   notes, and project standing.

### Increment 4 — `0.5.0-rc.1`: feature freeze

1. Run formatting, lint, type checking, all tests, build, package smoke, Action
   pin checks, CI on Node 20/22 and all three operating systems, and CodeQL.
2. Install the exact candidate from npm in a clean repository and exercise
   validate, inspect, analyze, generate, and verify.
3. Confirm v0.4 contracts produce byte-identical generated output.
4. Accept only release-blocking fixes after the RC tag.

### Stable promotion — `0.5.0`

Promote the unchanged validated candidate to npm `latest`, create the GitHub
Release with both assets, verify provenance and a clean install, then mark the
roadmap milestone complete.

## Verification matrix

| Surface       | Required evidence                                                                             |
| ------------- | --------------------------------------------------------------------------------------------- |
| JSON Schema   | Valid minimal/full blocks; every malformed shape and length boundary rejected.                |
| Normalization | Deterministic output; meaningful list order preserved; paths normalized safely.               |
| Semantics     | Duplicate, absolute, escaping, and invalid-extension paths rejected with stable diagnostics.  |
| Analyze       | Existing and missing decision/context files; read failures; JSON and human output.            |
| Adapters      | All five outputs, Markdown-special text, architecture-only, agents-only, and combined blocks. |
| Compatibility | Existing v0.4 contracts byte-identical; v2 corpus reproducible.                               |
| Distribution  | Package smoke, OIDC publish, provenance, release assets, and clean npm install.               |

## Completion criteria

- ADR-0032 and ADR-0033 are implemented and marked accordingly.
- All v0.5 roadmap exit criteria are checked.
- No v0.4 fixture changes without an explicit compatibility explanation.
- CI, CodeQL, package publication, and clean-install verification are green.
- `npm install -D @adameddahmouni/agent-ready` resolves to `0.5.0` after stable
  promotion; prereleases remain on `next`.
