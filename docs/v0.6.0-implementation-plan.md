# v0.6.0 implementation plan

**Status: planned; implementation not started.**

## Outcome

v0.6.0 makes verification evidence suitable for agent-to-human handoff and
closes two execution gaps: commands can declare their own bounded timeout, and
verification can reject stale generated instruction files before running
repository commands.

The release is additive. The contract remains `version: 1`, existing contracts
and verification records remain valid, and existing `verify --execute` behavior
does not change unless a new contract field or CLI flag is used.

## Accepted scope

- ADR-0034: an optional, validated handoff object supplied with
  `verify --execute --handoff <path>` and included in recorded evidence.
- ADR-0035: an optional `commands.<name>.timeout` integer from 1 through 3600
  seconds that overrides the CLI timeout for that command.
- ADR-0036: `verify --execute --check-generate`, which checks generated-file
  drift before executing any declared verification command.
- Stable diagnostics, JSON output, documentation, and `explain` entries for all
  new failure modes.
- Backward-compatibility fixtures for contracts, evidence records, CLI output,
  and generated adapters.

## Explicit non-goals

- Verifying whether handoff claims are true; only structure and limits are
  validated.
- A `handoff` contract block, automatic handoff authoring, Git-diff inference,
  or LLM-generated summaries.
- Per-command environment variables, working directories, shell selection, or
  network policy.
- Running verification commands after generated-file drift is detected.
- Contract `version: 2` or evidence-format migration of existing records.

## Design and dependency order

1. Accept ADR-0034, ADR-0035, and ADR-0036 before changing public surfaces.
2. Land the per-command timeout first because it is isolated and exercises the
   contract/schema compatibility path.
3. Land handoff validation and evidence serialization next, keeping parsing as
   a pure, independently tested boundary.
4. Land generate-drift integration last, reusing the existing generate check
   implementation rather than spawning the CLI recursively.
5. Freeze public names and diagnostic codes at beta; only compatible fixes are
   allowed afterward.

## Implementation sequence

### Increment 1 — `0.6.0-alpha.1`: per-command timeout

1. Write ADR-0035 and reconcile ADR-0006 and ADR-0014 with the narrowly
   reopened timeout decision.
2. Add `timeout` to the command JSON Schema, raw types, normalized types, and
   inspect output without changing contracts that omit it.
3. Resolve each command's timeout as `command.timeout ?? --timeout ?? 900` and
   pass the resulting milliseconds to the existing command runner.
4. Preserve the selected timeout in per-command evidence so a record explains
   the execution bound that was applied.
5. Test the 1/3600 boundaries, non-integers, precedence, mixed command
   timeouts, timeout termination, and unchanged legacy behavior.

### Increment 2 — `0.6.0-alpha.2`: structured handoff evidence

1. Write ADR-0034 and define a versionable internal handoff type with required
   `summary`, `filesChanged`, `commandsRun`, `assumptions`, `knownIssues`, and
   `requiresManualReview` fields.
2. Add a bounded JSON reader that rejects malformed JSON, wrong shapes,
   missing fields, unknown fields, summaries over 2000 characters, and array
   entries over 500 characters.
3. Add `--handoff <path>` to `verify --execute`; validate it even when
   `--record` is absent and include it only when a record is written.
4. Keep evidence deterministic apart from the existing timestamp and preserve
   the current record shape when `--handoff` is omitted.
5. Add `HANDOFF_FILE_INVALID` and `HANDOFF_FIELD_TOO_LONG` to diagnostics,
   JSON output, documentation, and the `explain` registry.
6. Test unreadable files, malformed JSON, every field boundary, Unicode,
   path handling, record/no-record behavior, and legacy evidence snapshots.

### Increment 3 — `0.6.0-beta.1`: generated-file drift gate

1. Write ADR-0036 and extract/reuse the generate-check domain function behind
   both `generate --check` and verification.
2. Add `--check-generate`, valid only with `verify --execute`.
3. Run the drift check before the first declared command; on drift, emit the
   existing generate diagnostics, write evidence when requested, and execute
   zero repository commands.
4. Represent the preflight result explicitly in evidence without pretending it
   is a user-declared command.
5. Test clean output, drift, missing generated files, adapters selected by the
   contract, JSON/human output, record behavior, and proof that commands are
   skipped on failure.
6. Update the composite Action inputs only if ADR-0036 explicitly includes
   parity; otherwise document CLI-only scope for v0.6.0.

### Increment 4 — `0.6.0-rc.1`: integration and feature freeze

1. Exercise all combinations of handoff, record, timeout, check-generate, and
   command success/failure/timeout.
2. Update the contract, CLI, evidence, diagnostics, CI integration, security,
   project-standing, and upgrade documentation.
3. Mark ADRs and roadmap checkboxes implemented only after tests and docs agree
   with the shipped behavior.
4. Run formatting, lint, type checking, tests, build, package smoke, Action pin
   checks, Node 20/22 CI on Linux/macOS/Windows, and CodeQL.
5. Install the exact candidate in a clean fixture repository and exercise the
   three new surfaces through the packaged CLI.
6. Accept only release-blocking fixes after the RC tag.

### Stable promotion — `0.6.0`

Promote the unchanged validated candidate to npm `latest`, create the GitHub
Release and assets, verify provenance and a clean install, and then mark the
v0.6.0 roadmap milestone complete.

## Verification matrix

| Surface       | Required evidence                                                                                    |
| ------------- | ---------------------------------------------------------------------------------------------------- |
| Schema        | Timeout accepts only integers 1–3600; contracts without it remain valid.                             |
| Precedence    | Command timeout overrides CLI timeout; CLI timeout overrides 900-second default.                     |
| Handoff       | Every required field, type, size boundary, read failure, and malformed file is covered.              |
| Evidence      | Legacy shape is unchanged when flags are absent; new metadata is deterministic and documented.       |
| Drift gate    | Clean, changed, and missing generated files; no declared command runs after preflight failure.       |
| CLI           | Human and JSON output, invalid flag combinations, exit codes, and help text.                         |
| Compatibility | Existing contract, evidence, and adapter fixtures remain unchanged unless explicitly justified.      |
| Distribution  | Package smoke, clean packaged-CLI fixture, npm provenance, release assets, and install verification. |

## Completion criteria

- ADR-0034, ADR-0035, and ADR-0036 are accepted and implemented.
- All v0.6.0 roadmap exit criteria are checked with links to tests or release
  evidence.
- `verify --execute --record --handoff` writes a documented structured handoff.
- Per-command timeout precedence works across supported operating systems.
- `verify --execute --check-generate` detects drift and executes no declared
  command on preflight failure.
- Existing contracts and evidence consumers require no migration.
- CI, CodeQL, package publication, provenance, and clean-install verification
  are green; npm `latest` resolves to `0.6.0`.
