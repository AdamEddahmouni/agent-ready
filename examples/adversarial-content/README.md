# adversarial-content

Example repository used to exercise Phase 8's Markdown-escaping behavior:
`project.name`/`description` and `command.description` containing
block-starting/inline-significant characters, a `command.run` containing
backticks, a glob pattern containing a backtick, and an
`instructions.sources` path containing a space and parentheses. Every
field here is schema-valid; see
[docs/decisions/0017-adapter-output-markdown-escaping.md](../../docs/decisions/0017-adapter-output-markdown-escaping.md).
